lti:ServiceProfile rdf:type owl:Class ; 
  rdfs:comment """This entity defines a profile for a service.  It is a \"profile\" in the sense that it specifies a subset of operations that are declared by the full  interface for some service.  For example, a profile might list the \"read\" operations of a service but exclude the \"write\" operations.  This is useful because a product (Tool or TC) might want to offer the read-only operations to a partner, but it might not want to expose write operations.  (On the other hand, it is perfectly legitimate for a profile to list all of the operations that are declared in the service interface if desired.)
ServiceProfiles are used for four different purposes: (1) they allow a product to advertise the operations that it offers to a partner, (2) they allow a product to declare the operations that it intends to use, (3) they allow a product to advertise the security profiles that are available for service operations, and (4) they allow a product to assert the security profiles that will be used for service operations. """ .
